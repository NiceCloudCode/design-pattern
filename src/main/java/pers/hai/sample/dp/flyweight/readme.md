## Flyweight Design Pattern.

> 享元模式：运用共享技术有效地支持大量细粒度的对象。

### 应用场景

> 如果一个应用使用了大量的对象，而大量的这些对象产生了很大的存储开销时就应该考虑使用；还有就是对象的大多数状态可以外部状态，如果删除对象的外部状态，那么可以用相对较少的共享对象取代很多组对象，此时可以考虑使用享元模式。
> 
> 1. 字符串的对象池
